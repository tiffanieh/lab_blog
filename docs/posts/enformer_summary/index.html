tiffanie

paper link

objective: improve gene expression prediction accuracy from DNA sequences using a type of DL architecture (Enformer) that can include info from long-range interactions in genome → higher accuracy for variant effect predictions on gene expression, learned to predict enhancer-promoter interactions directly from DNA sequence → future steps: more effective fine-mapping of human disease associations

Main

Population-based association studies+models predicting gene expression+chromatin states from DNA sequences:

  • the former limited to common variants; hard to disentangle causality from assoc due to LD (linkage disequilibrium)
  • also hard to test all variants of interest in different relevant contexts in experimental validation
  • sequence-based computational models have limited accuracy

Currently: deep CNNs to predict gene expr from DNA seqs for human+mouse genomes

  • drawback: can only consider seq elements < 20 kb away from TSS (transcription start site) b/c locality of convolutions limits info flow in net between far apart elements
    • many regulatory elements can infleunce gene expr from far away so need to increase info flow between distal elements

Results

Solution: Enformer — a new model architecture to predict gene expr+chromatin states in humans+mice from DNA seqs

  • transformers: a class of DL models for NLP; recently applied to model short DNA seqs
    • consist of attention layers that transform each position in input sequence by computing weighted sum across representations of all other positions in sequence
    • attention weight between any 2 positions depends on embeddings of current representation vectors+distance between
    • model can refine prediction at a TSS by getting info from relevant regions (e.g. relevant enhancers)
    • better info flow between distal elements since each position directly attends to all other positions in sequence
  • vs convolutional layers — need many many layers to get to far away elements due to their local receptive field
  • transformer layers → increase receptive field → reach distal regulatory elements up to 100 kb away
    • expands # relevant enhancers seen by model from 47% to 84% (<100 kb)
  • Enformer better at predicting RNA expr at TsS of human protein-coding genes, consistent across all 4 types of genome wide tracks (CAGE — transcriptional activity, histone modifications, TF binding, DNA accessibility)
    • esp CAGE — prob b/c tissue-specific gene expr depends a lot on distal elements
  • greater predictive accuracy than ExPecto (model trained to predict gene expr levels measured by RNA seq) for across-genes+across-tissues

Demonstrating benefit of attention layers

  • (compared to dilated convolutions in previous model)
  • replace attention layers w/ dilated convolutions+tuned learning rate for optimal performance → attention layers outperformed dilated convolutions across all model sizes, # layers, # training data pts
  • observed performance drop when restricting receptive field of Enformer to that of the previous model by replacing global attention layers w/ local
  • increase # params → improve model performance (consistent w/ recent advances in NLP)
  • use custom relative positional basis fns in transformer layers (distinguish between proximal and distal regulatory elements more easily)
    • noticeable performance improvement compared to typically used relative basis fns+absolute positional encodings in NLP lit

Enformer attends to cell-type-specific enhancers

  • what seq elements Enformer uses to make predictions? → compute 2 diff gene expr contribution scores:
    1. input gradients (gradient \(\times\) input)
    2. attention weights
    for several genes w/ CRISPRi-validated enhancers

contribution scores — highlight most predictive input seq for particular gene’s expression

  • input gradients are tissue- or cell-type-specific (b/c computed wrt particular output CAGE sample)
  • vs attention weights — internal to model+shared among all tissue+cell-type predictions
  • observed that contribution scores of several genes correlated w/ histone H3 acetylated at K27 (H3K27ac) and highlighted not just local promoter regions but also distal enhancers >20 kb away
    • suggests that Enformer is looking at biologically relevant regions when making predictions; gene expr contribution scores could be used to prioritize relevant enhancers
  • unsolved problem: how to link candidate enhancers id’ed vai biochemical annotations to target genes
    • issue: computational models have historically low accuracy b/c of noisy labels+class imbalance
  • evaluate ability of contribution scores to pinpoint relevant enhancers for a specific gene: compare several contribution scores across all tested enhancer-gene pairs in 2 large-scale CRISPRi studies performed on K562 cell line
  • CRISPRi used to suppress activity of > 10,000 candidate enhancers+measure effect on gene expr
  • contribution scores prioritize validated en-gene pairs w/ higher accuracy despite only using DNA seq as input — never trained to explicity locate enhancers
    • can be used for arbitrary seq variations lacking experimental data
  • model uses diff enhancer seqs in diff cell types as expected → can prioritize candidate enhancers in cell types used for model training

has model learned about insulator elements (separate 2 topologically associating domains, i.e. TADs, minimize enhancer-promoter crosstalk between the two)

  • used attention matrices (more efficient than input gradients to compute b/c of many output targets) of seqs centered at TAD boundaries+compared to attention from seqs w/ no particular alignment
  • Enformer paid more attention to TAD boundaries than random positions; less attention to regions on opposite side of boundary — consistent w/ reduced inter-TAD interactions in biology
  • suggest that model has not only learned about role of tissue-specific enhancers+promoters, but also about insulator elements (and their role in inhibiting info flow between genomic compartments)

Enformer improves variant effect prediction on eQTL data

  • central goal: predict influence of genetic variants on cell-type-specific gene expr
    • → inform fine-mapping of many noncoding assoc w/ phenotypes of interest from GWAS
  • computational models that predict regulatory activity from DNA seqs can process distinct alleles+compare preds to score genetic variants
  • successful model can produce eQTL results w/o having to measure lots of individual gene expr profiles
  • challenge: LD in profiled pop → transfers causal eQTL’s effect to nearby co-occurring variants’ measurements
  • SLDP (signed linkage disequilibrium profile) regression: measures genome-wide statistical concordance between signed variant annotations (e.g. model preds) and GWAS summary stats (e.g. GTEx eQTLs) while accounting for LD
  • Enformer preds for noncoding-variant activity seem to improve for samples w/ similar cell-type composition mostly

how to assess use of preds for identifying causal variants → classification task per tissue → discriminate likely causal variants from likely spurious eQTLs

  • each variant represented by its pred diff vector (reference-alternative allele, summed across seq) for all 5313 human datasets
  • trained random forest classifiers
  • preds enabled more accurate classifier for most GTEx tissues

Enformer improves MPRA mutation effect prediction

  • eval Enformer’s performance on second, independent variant effect pred tas w/ dataset where MPRAs (massively parallel reporter assays) directly measured fnal effect of genetic variatns through sutration mutagenesis of several enhancers+promoters in variety of cell types
  • for each variant, eval effect as pred diff between ref+alt allele → retrieve 5313 features
  • compare 2 approaches:
    1. train lasso reg model on provided training set for each gene
    2. preselect subset of features corresponding to cell-type-matched/agnostic preds of changes in CAGE+DNase → generate summary stat of features

Lasso regression w/ Enformer preds as features had best average correlation across all loci

Future steps

  • systematically apply Enformer to fine-map existing GWAS studies
  • prioritize rare or de novo variants for rare disorders
  • impute regulatory activity across species to study cis-regulatory evolution

Methods

Model architecture

diagram of typical Transformer architecture:

Enformer architecture has 3 parts

  1. 7 convolutional blacks w/ pooling
  2. 11 transformer blocks
  3. a cropping layer followed by final pointwise convolutions branching into 2 organism-specific network heads

input: one-hot-encoded DNA seq

  • A = \(\left[1,0,0,0\right]\)
  • C = \(\left[0,1,0,0\right]\)
  • G = \(\left[0,0,1,0\right]\)
  • T = \(\left[0,0,0,1\right]\)
  • N = \(\left[0,0,0,0\right]\)

of length 196,608 bp → predicts 5313 genomic tracks for human genome, 1643 tracks for mouse genome (each of length 896 corresponding to 114,688 bp aggregated into 128-bp bins)

  • convolutional blocks w/ pooling first reduce the spatial dimension from 196,608 bp to 1526 so each seq position vector represents 128 bp
  • transformer blocks capture long-range interactions across seq
  • cropping layer trims 320 positions on each
  • to avoid computing loss on far ends b/c these regions are disadvantage (observe regulatory elements only on one side, toward seq center)
  • 2 output heads predict organism-specific tracks

changes: use transformer blocks instead of dilated convolutions, attention pooling instead of max pooling, twice as many channels, 1.5x longer input seq

attention pooling: summarizes contiguous chunk of input sequence \(\mathbf{x}_{k:k+L_p}^{full}=\mathbf{x} \in R^{L_p\times C}\) across \(L_p\) positions for each of the \(C\) chanels

  • output value \(\mathbf{h} \in \mathbf{\mathit{R}}^C\):

\[ h_j = \frac{\sum_i exp(x_i \cdot w_j)x_{ij}}{\sum_i exp(x_i\cdot w_j)} \]

where i indexes seq position in pooling window, which is weighted by exponentiated dot product \(\mathbf{x}_i\cdot \mathbf{w}_j\) and \(\mathbf{\mathit{w}} \in R^{C\times K}\) is a matrix of learned weights

  • apply attention pooling to contiguous chunks of original input seq using window size \(L_p = 2\) and stride of 2

initialize \(\mathbf{w}\) to \(2\times \mathbf{1}\)

  • \(\mathbf{1}\) = identity matrix
  • prioritizes larger value → similar operation to max pooling
  • slightly better performance than random initialization or initialization w/ zeros (average pooling)

MHA (multi-head attention) layers used to capture long-range interactions across sequence

  • each head has separate weights for queries, keys, values that transform input seq
    • separate set of weights \(\mathbf{w}^q \in \mathbf{R}^{C\times K}\), \(\mathbf{w}^k \in \mathbf{R}^{C\times K}\), \(\mathbf{w}^v \in \mathbf{R}^{C\times K}\)
    • → transform input sequence \(\mathbf{x}\in \mathbf{R}^{L\times C}\) into:
      • queries \(\mathbf{q}_i=\mathbf{x}_i\mathbf{w}^q\)
      • keys \(\mathbf{k}_j=\mathbf{x}_j\mathbf{w}^k\)
      • values \(\mathbf{v}_j=\mathbf{x}_j\mathbf{w}^v\)
  • queries = current info at each position
  • keys = info each position looks for to attend to
  • dot product between queries+keys plus relative positional encodings \(\mathbf{R}_{ij}\) → attention matrix
    • computed as \(\mathbf{a}_{ij} = softmax(\mathbf{q}_i\mathbf{k}_j^T/\sqrt{K}+\mathbf{R}_{ij})\)
    • entry \(a_{ij}\) = amount of weight that query at position i puts on key at position j
  • values = info propagated to positions attending to a specific position
  • each single attention head computes its output as weighted sum across all input positions: \(\mathbf{av}\)
    • lets query position use info across whole seq
    • multiple heads compute w/ independent parameters
  • outputs from each head concatenated to form final layer output followed by linear layer to combine

Enhancer prioritization:

  1. Gradient \(\times\) input: compute abs value of gradient of CAGE targets at TSS w. regard to input reference seq nucleotide
    1. equivalent to computing gradient \(\times\) input attirbutions (since one-hot-encoding)
    2. CAGE at TSS = summing abs grad values from 3 adjacent bins (including bin overlapping TSS, one flanking bin on each side)
    3. enhancer-gene scare obtained by summing abs grad\(\times\)input scores in 2-kb window centered at enhancer
  2. attention: averaged transformer attention matrices across all heads+layers → extract row corresponding to query index positioned at TSS
    1. keys correspond to diff spatial positions
    2. attention values = how much model attended to these positions when making preds for TSS
    3. en-gene score — sum attention scores in 2-kb window centered at enhancer
  3. ISM: in silico mutagenesis en-gene score
    1. computed by comparing K562 CAGE preds at TSS from ref seq with preds from modif seq where 2-kb enhancer seq was replaced by a random sequence: \(|f(modified)-f(reference)|\)
